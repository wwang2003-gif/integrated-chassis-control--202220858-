## 1\. 설계 개요 (1 페이지)

1-2 문단으로:

본 과제의 목표는 차량의 횡방향, 종방향, 그리고 수직방향 동역학을 통합적으로 제어하여, 급제동이나 극한의 선회 주행과 같은 비정상 주행 상황에서도 타이어의 물리적 마찰 한계를 초과하지 않고 차량의 조종 안정성과 승차감을 동시에 확보하는 것이다. 이를 구현하기 위해 본 설계에서는 복잡한 비선형 최적화 기법보다는 실시간 연산 안정성과 제어기 튜닝의 직관성이 검증된 제어 기법들을 유기적으로 조합하였다. 구체적으로는 차량의 속도 변화에 따른 거동 특성을 보상하기 위해 '속도 적응형 PI 제어(Gain-Scheduled PI Control)'를 주축으로 사용하였으며, 한계 상황에서의 안전성을 확보하기 위해 ESC(Electronic Stability Control) 및 ABS(Anti-lock Braking System)에는 '임계값 기반 비례 제어(Threshold-based Proportional Control)'를, 승차감 개선을 위해서는 '연속형 스카이훅(Continuous Skyhook) 제어'를 적용하였다.



이러한 제어 기법을 선택한 핵심적인 이유는 차량 제어 시스템이 요구하는 강건성(Robustness)과 운전자의 조향 의도 존중이라는 설계 철학 때문이다. Rajamani의 저서 Vehicle Dynamics and Control 및 다수의 차량 동역학 교과서에 따르면, 고차원적인 비선형 제어기보다 Gain-scheduled PID 계열의 제어기가 모델의 불확실성에 대한 방어력이 뛰어나며, 실제 양산 차량 적용 시 안정성과 실시간 연산 부하 측면에서 월등한 효율성을 보인다. 또한, 베타-limiter(사이드 슬립 제한)와 ABS 개입 방식은 타이어 마찰원(Friction Circle) 한계에 도달하는 순간에만 강력하게 역방향 제동력을 인가하는 방식을 택하였다. 이는 평상시 운전자의 조향 및 제동 의도를 최대한 방해하지 않으면서도, 락업이나 스핀이 발생하려는 찰나에만 개입하여 타이어의 피크 마찰력을 유지시키는 가장 현실적이고 효율적인 제어 논리이기 때문이다. 결론적으로 본 설계는 이론적 타당성과 실무적 구현 가능성을 최적의 제어 지향하였다.  

각 제어기 한 줄 요약:

* **ctrl\_lateral**: 속도 적응형 PI 제어 방식으로 yaw rate 추종 + 임계값 기반 비례 제어으로 β-limiter
* **ctrl\_longitudinal**: Anti-windup 기법이 적용된 PI 제어 기반으로 속도를 추종하며, 4채널 ABS를 위한 타이어 슬립율 피드백 연산을 수행한다.
* **ctrl\_vertical**: 차체 절대 속도와 휠 상대 속도를 바탕으로 한 연속형 스카이훅 제어를 적용하였다.
* **ctrl\_coordinator**: <yaw moment → brake 분배 방식>: 요 모멘트 기반 좌우 차동 제동을 분배하며, 4륜 독립 슬립 초과량에 따른 비례 감압(ABS) 제어를 수행한다.

## 2\. 수학적 모델링 (1-2 페이지)

### 2.1 사용한 plant 단순화

어떤 모델을 제어 설계에 사용했는가? (bicycle? 3DOF?) 학생은 14DOF plant 위에 검증하지만, **제어기 설계** 자체는 보통 더 단순한 모델 위에서 한다.



&#x20; 제어기 설계를 위해 14DOF의 복잡한 비선형 차량 모델(Full Vehicle Model) 대신, 횡방향 동역학 특성을 직관적으로 해석할 수 있는 2자유도 자전거 모델(2-DOF Bicycle Model)을 기준 플랜트로 선정하여 단순화하였다. 종방향과 수직방향(Roll, Pitch, Bounce)의 동역학적 결합(Coupling)을 배제하고, 좌우 바퀴를 축약하여 중심선 상의 하나의 앞바퀴와 뒷바퀴로 표현하였다. 실제 검증은 시뮬레이터의 14DOF 환경에서 수행하지만, 제어기 설계 단계에서는 이 단순화된 모델을 통해 횡방향 조향 입력에 따른 요 레이트(Yaw Rate)와 횡방향 속도 변화의 선형적 인과관계를 정의하고 연산 효율성을 확보하였다.



### 2.2 State-space 표현

$$\\dot{x} = Ax + Bu, \\quad y = Cx + Du$$

상태 변수, 입력, 출력 정의 + A, B 행렬 표현. Bicycle Model 사용 시:
$$x = \[v\_y, r]^T, \\quad u = \\delta$$
$$\\dot{v}\_y = -(C\_f + C\_r)/(mV\_x),v\_y + ((l\_r C\_r - l\_f C\_f)/(mV\_x) - V\_x),r + C\_f/m,\\delta$$
$$\\dot{r} = (l\_r C\_r - l\_f C\_f)/(I\_z V\_x),v\_y - (l\_f^2 C\_f + l\_r^2 C\_r)/(I\_z V\_x),r + l\_f C\_f/I\_z,\\delta$$



횡방향 속도()와 요 레이트()를 상태 변수로 조향각을 제어 입력으로 하는 상태 공간 방정식은 다음과 같이 정의된다.

\-상태 변수 (State vector): 

\-입력 (Input, u) =  (전륜 조향각)

\-출력 (Output), y=r

이를 형태의 행렬식으로 표현하면 아래와 같다. 



$$\\dot{v}\_y = -\\frac{C\_f + C\_r}{mV\_x} v\_y + \\left( \\frac{l\_r C\_r - l\_f C\_f}{mV\_x} - V\_x \\right) r + \\frac{C\_f}{m} \\delta$$

$$\\dot{r} = \\frac{l\_r C\_r - l\_f C\_f}{I\_z V\_x} v\_y - \\frac{l\_f^2 C\_f + l\_r^2 C\_r}{I\_z V\_x} r + \\frac{l\_f C\_f}{I\_z} \\delta$$



### 2.3 가정 + 한계

설계된 수학적 모델은 다음과 같은 가정과 물리적 한계를 내포하고 있다.



&#x20;일정 종속도 가정: 횡방향 제어기 설계 시점에서는 종방향 속도가 상수로 유지된다고 가정하여 횡방향/종방향 제어를 분리하였다. 급가감속 시에는 이 가정이 깨지므로 오차가 발생할 수 있다. 



선형 타이어 모델 즉, 소슬립 영역에서는 타이어의 횡력이 슬립 앵글에 비례한다는 선형 구간 내에서의 거동만을 가정하였다. 한계 주행 시 발생하는 비선형적인 마찰 포화는 반영되지 않는다.



미소 조향각 가정에서는 전륜 조향각이 매우 작다고 가정하여 로 근사화하였다.



하중 이동 무시는 롤링 및 피칭 모션을 무시하였기 때문에 선회나 제동 시 발생하는 전후/좌우 윤하중 이동이 타이어 횡력에 미치는 영향이 설계 모델에는 반영되지 않는 한계가 있다.

## 3\. 제어기 설계 (3-4 페이지)

### 3.1 ctrl\_lateral — AFS + ESC

**설계 목표**:

본 제어기의 1차적 목표는 운전자의 조향 각도로부터 계산된 목표 요 레이트를 추종하여 차량의 선회 성능을 극대화하는 것이다. 동시에 차량의 횡방향 운동 한계를 나타내는 사이드 슬립 앵글을 물리적 임계치 이내로 제어하여, 극한 선회 상황에서의 스핀(Spin)을 방지하고 조종 안정성(Handling Stability)을 확보하는 것을 목적으로 한다. 정량적 성능 지표로 정착 시간(Settling time) 0.8s 미만, 오버슈트(Overshoot) 10% 미만을 설정하여 응답의 신속성과 안정성을 동시에 충족하고자 하였다. 

**선택 기법**:  속도 적응형(Gain-Scheduled) PID 제어기

**Gain 계산 과정**:
<예시>

PID 의 경우 Ziegler-Nichols / IMC tuning 사용:

\-1차 모델 근사: 차량의 요 레이트 전달함수를 기본 모델 $G(s) = \\frac{K}{\\tau s + 1}$로 설정하였다. 여기서 $K$는 조향 게인, $\\tau$는 차량의 응답 시정수이다.

\-PID 튜닝: Ziegler-Nichols 기반으로 기초 이득을 도출하였다. 응답의 신속성을 위해 $K\_p$를 설정하였고, 잔류 오차 제거를 위한 적분기 $K\_i$를 추가하였다. 특히, 횡방향 오차의 급격한 변화를 감쇄시키기 위해 미분기 $K\_d$를 병행 적용하였다.

&#x20;ZN Tuning 공식:

\-$K\_p = 0.6 / (K \\cdot \\tau)$

\-$K\_i = K\_p / (0.5 \\cdot \\tau)$

\-$K\_d = K\_p \\cdot (0.125 \\cdot \\tau)$

\-속도 적응형 게인 스케줄링 (Gain Scheduling):

차량의 속도 $V\_x$에 따라 제어 게인이 가변되도록 설계하였다. $f(V\_x) = \\min(V\_x / V\_{ref}, 2.0)$ 함수를 게인에 곱함으로써, 저속에서는 응답성을 높이기 위해 게인을 증대시키고, 고속에서는 오버슈트를 억제하기 위해 게인을 최적화하였다. 이는 ctrl\_lateral.m 코드 내 f\_vx 변수를 통해 구현되었다.



**최종 게인 + 정당화**:

```matlab
CTRL.LAT.Kp = 속도 스케줄링 변수(f(vx)와 결합하여 고속 선회 시에도 조향 응답이 저하되지 않도록 보정하였다.

CTRL.LAT.Ki = 적분 오차 누적을 방지하기 위해 CTRL.LAT.intMax로 클램핑(Anti-windup)을 적용하여 제어기의 안정성을 확보하였다.
CTRL.LAT.Ki = 차량이 차선 변경과 같은 급격한 조향 기동을 수행할 때, PID 제어기의 비례(P) 항만으로는 조향각의 오버슈트(Overshoot)를 억제하는 데 한계가 있다. 미분(D) 항은 오차의 변화율($\\frac{de}{dt}$)에 반응하므로, 목표 경로를 추종하는 과정에서 발생하는 요 레이트의 과도한 거동을 즉각적으로 억제하여 차량의 자세가 목표치에 도달할 때 진동 없이 안정적으로 정착(Settling)하도록 돕는다

BETA\\\_THRESHOLD = deg2rad(3), 약 3도로 설정하였다. 횡방향 물리적 한계치의 90% 수준에서 ESC가 개입하도록 하여 운전자의 초기 조향 의도를 최대한 존중하면서도 스핀 발생 시에만 즉각적으로 역방향 요 모멘트Mz를 생성하여 자세를 회복시킨다.

BETA\\\_GAIN = 8000으로 설정하였다. 이는 슬립 앵글이 임계치를 벗어날 때, 벗어난 정도에 비례하여 강력한 복원 모멘트를 인가함으로써 차량이 선회 경로를 이탈하지 않도록 강건한 자세 제어를 수행한다. 
```

### 3.2 ctrl\_longitudinal — 속도 + ABS

**설계 목표**: 본 제어기는 차량의 목표 종방향 속도Vx,ref를 추종하는 순항 제어(Cruise Control) 기능과, 급제동 시 타이어의 락업(Lock-up)을 방지하여 최대 제동력을 확보하는 ABS(Anti-lock Braking System) 기능을 통합적으로 수행하는 것을 목표로 한다. 특히 B1 시나리오와 같은 급제동 상황에서 노면 마찰력을 최대한 활용하여 제동 거리 66.5m 이내의 안정적인 정지를 달성하고자 한다. 





**선택 기법**:  속도 추종 PI 제어기

**Gain 계산 과정**:

속도 추종 (PI 제어): 차량의 가속 및 감속을 조절하기 위해 Anti-windup이 포함된 PI 제어기를 선택하였다. 제어 오차(Vx,ref - Vx)를 적분하여 정상 상태 오차를 제거하며, 제어기 출력의 포화를 방지하기 위해 CTRL.LON.intMax를 설정하여 Windup 현상을 억제하였다.



ABS (4채널 독립 비례 제어): 단순한 뱅뱅(Bang-bang) 제어의 한계를 극복하기 위해, 각 바퀴의 슬립율을 개별적으로 연산하는 4채널 독립 비례 제어를 설계하였다. 이는 특정 바퀴가 잠기는 순간 해당 휠의 브레이크 압력만을 선택적으로 제어하여, 차량 전체의 제동 효율을 극대화하는 방식이다.



**최종 게인 + 정당화**:



CTRL.LON.Kp, Ki: 차량의 질량과 요구 가속도 한계(LIM.MAX\_AX)를 고려하여 튜닝하였다. 급제동 시 PI 제어기의 반응 지연을 보상하기 위해 error\_vx < -5.0 조건에서 즉각적으로 최대 브레이크 토크를 인가하도록 강제하는 예외 처리 루틴을 포함하였다.



kappa\_target = 0.098: 타이어의 마찰 특성 곡선상 마찰력이 최대가 되는 지점이 약 10% 슬립 부근임을 고려하여 임계치를 설정하였다. 0.098은 absSlipRMS의 만점 기준인 0.100에 가장 근접하면서도 락업을 방지할 수 있는 최적의 안전선이다.



K\_abs = 15.0: 슬립율이 목표치를 초과할 경우, 초과분에 비례하여 브레이크 토크를 감압하는 상수이다. 이 값은 브레이크를 너무 급격하게 풀지 않으면서도 미끄러짐을 빠르게 억제할 수 있는 최적치로 설정되었다.



### 3.3 ctrl\_vertical — CDC (있다면)

설계 목표: 본 제어기는 차량의 승차감을 향상시키고 노면 입력에 대한 차체의 지면 추종성을 최적화하기 위해 각 휠의 감쇠력을 실시간으로 조절하는 CDC 시스템을 구현하는 것이다. 차체의 바운싱 및 휠 호핑 모드를 제어하여 노면으로부터 유입되는 진동을 효과적으로 감쇄시키는 것을 목적으로 한다. 



선택 기법: 다양한 준능동(Semi-active) 제어 기법 중 구현의 직관성과 제어 효율이 검증된 연속형 스카이훅(Continuous Skyhook) 제어를 선택하였다. 스카이훅 제어는 차체(Sprung mass)를 절대 좌표계상의 가상 천장에 매달린 것처럼 정지시키려는 이론적 배경을 가지며, 댐퍼가 차체의 수직 운동 방향과 반대 방향으로 힘을 가함으로써 진동 에너지를 소산시킨다.



제어 로직 및 정당화:에너지 소산 조건: 차체 속도와 댐퍼의 상대 속도의 곱이 양수인 경우에만 댐퍼가 에너지를 소산시키고 있다고 판단하여 능동적으로 개입한다.



감쇠력 산출: C\_sky = skyGain (z\_s / z\_s - z\_u) 수식을 통해 노면 상태에 맞게 감쇠 계수를 실시간으로 연속 가변시킨다. 이때 0으로 나누는 오차를 방지하기 위해 매우 작은 값(epsilon = 10^{-6}을 추가하여 안정성을 확보하였다.





감쇠력 제한: 계산된 C\_sky 값은 댐퍼의 물리적 하드웨어 한계를 고려하여 CTRL.VER.cMin(500 Ns/m)에서 CTRL.VER.cMax(5000 Ns/m) 사이로 클리핑하였다.



기법 정당화:이 기법은 고주파수 노면 입력에 대해 감쇠력을 낮게 유지하여 타이어의 지면 추종성을 높이고 승차감을 확보하며, 저주파수 영역의 차체 바운싱 발생 시에는 감쇠력을 높여 제어력을 극대화한다. 본 과제의 14DOF 플랜트에서 발생하는 복합적인 진동 모드를 효율적으로 억제하기에 가장 적합한 제어 전략이며, 별도의 고가 액추에이터 없이 기존 댐퍼의 밸브 개도 조절만으로 구현이 가능하여 경제성 측면에서도 우수하다.

### 3.4 ctrl\_coordinator — Actuator Allocation

설계 목표: 본 제어기는 상위 제어기(Lateral, Longitudinal, Vertical Controller)로부터 하달된 요 모멘트(Mz), 총 종방향 힘Fx,total, 그리고 각 휠의 감쇠 계수ci 명령을 실제 차량의 액추에이터(4륜 브레이크 토크, 조향각, 가변 댐퍼 계수)가 수행 가능한 물리량으로 변환하여 최종 배분하는 역할을 수행한다. 단순히 명령을 전달하는 것을 넘어, 차량의 물리적 한계를 고려한 제약 조건 하에서 동역학적 성능을 최적화하는 것이 핵심이다.



분배 방식 및 논리

종방향 제동 토크 배분 (60:40 비율):급제동 시 발생하는 차량의 하중 이동(Load Transfer) 특성을 반영하여 전후륜 제동력을 60:40의 비율로 기본 배분하였다. 이는 정적 상태와 동적 상태 모두에서 타이어의 지면 수직 하중(Fz)과 제동력의 합이 최적의 마찰원(Friction Circle) 내에 위치하도록 유도하여, 어느 한쪽 차축이 조기에 잠기는 현상을 방지하고 차량의 직진 안정성을 확보한다.



ESC 요 모멘트 차동 배분 (Differential Braking):ESC로부터 요청된 요 모멘트Mz는 차량의 선회 방향과 반대되는 복원 모멘트를 생성하기 위해 좌우 휠의 브레이크 토크 차이를 통해 구현된다.



전륜 차동 토크: 



후륜 차동 토크: 



이 수식은 차량의 윤거(Track width)와 바퀴 반경(rw)을 고려하여 요청된 모멘트가 정확히 생성되도록 설계되었다.



4륜 독립 ABS 제어 (Independent P-Control): 급제동 시 앞바퀴와 뒷바퀴가 겪는 하중 조건이 극명하게 갈리는 점에 착안하여, 4륜 독립 ABS 로직을 구현하였다.



독립적 감압: 네 바퀴의 슬립율(kappa i)을 개별적으로 모니터링하여, 한계치(kappa(target)=0.095)를 초과하는 특정 휠의 브레이크 토크만을 비례 제어(P-Control)로 개별 감압한다.

Yaw Moment 인가 패턴: 제어기는 단순히 제동력을 균등 배분하는 것이 아니라, 차량의 관성 요 모멘트를 능동적으로 상쇄시키는 비대칭 차동 제동 전략을 취하였다. 특히 선회 중 제동으로 인해 뒤가 흐르는 오버스티어(Oversteer) 상황에서, ctrl\_coordinator.m은 조향 의도와 반대 방향의 요 모멘트를 생성하여 차량의 자세를 물리적으로 강제 복원하였다.



슬립율 기반 독립 제동력 분배: 본 제어기는 4개의 휠에 대해 독립적인 비례 제어(P-Control)를 수행하여, 슬립이 발생한 휠에 대해서만 선택적으로 제동력을 감압하였다. 이 과정에서 앞바퀴는 높은 마찰력을 유지하여 선회 조향력을 확보하고, 뒷바퀴는 슬립 억제를 통해 차량의 후미가 미끄러지는 현상을 방지하였다. 결과적으로 전후륜의 하중 이동을 고려한 동역학적 균형이 제어기의 핵심 성공 요인으로 작용하였다.



결론적으로 이번 설계는 차량의 동역학적 한계를 물리적으로 초과하기 직전에 ESC와 독립적 차동 제동을 결합하여 개입함으로써 극도의 불안정 상황에서도 차량이 경로를 이탈하지 않도록 자세를 정밀하게 복원하였다는 점에서 의미가 크다.

## 5. 분석 + 한계 (1-2 페이지)

### 5.1 가장 성공적이었던 시나리오
어느 시나리오에서 가장 큰 KPI 개선이 있었는가? 왜?



가장 성공적이었던 시나리오본 과제의 시나리오 중 가장 큰 폭의 KPI 개선이 나타난 것은 A7(Brake-in-turn) 시나리오이다. 베이스라인 제어기(OFF)에서 차량의 sideSlipMax는 30.48°에 달하여 선회 중 제동 시 차량이 스핀아웃(Spin-out)하는 불안정한 거동을 보였으나, 내가 설계한 통합 제어 시스템을 적용한 후(ON)에는 1.82°로 약 94.0%의 극적인 개선을 달성하였다.



\-가장 큰 개선이 있었던 이유



횡방향/종방향 통합 제어의 시너지: A7 시나리오는 선회 중 제동이라는 가장 복합적인 동역학적 조건이 요구된다. 이때 단순히 요 레이트만 추종하거나 제동력만 배분하는 개별 제어기는 한계에 부딪히게 된다. 본 시스템은 ctrl\_lateral이 횡방향 안정성을 유지하는 동시에, ctrl\_coordinator가 상위 제어기들의 명령을 통합하여 4륜 차동 제동(Differential Braking)을 수행함으로써 차량의 선회 궤적을 능동적으로 수정하였다.



한계 상황에서의 강건성: sideSlipMax 개선율이 압도적인 이유는 제어기가 차량의 물리적 한계점인 BETA\_THRESHOLD(3°)를 사전에 인지하고, 임계치를 넘어서기 직전 즉각적인 역방향 요 모멘트(M\_z)를 인가했기 때문이다. 이는 차량의 운동 에너지가 제어 불능 상태로 전이되는 것을 물리적으로 차단하는 가장 효과적인 전략이었다.



타이어 마찰력의 효율적 분배: 특히 급제동 시 발생하는 전후륜 하중 이동(Load Transfer)을 ctrl\_vertical과 ctrl\_coordinator가 사전에 감지하고 이를 반영하여 4륜의 브레이크 토크를 비대칭으로 재배분하였다. 이로 인해 타이어가 마찰원(Friction Circle) 내에서 최대의 횡력과 종력을 확보할 수 있었고, 결과적으로 선회 중 제동 상황에서도 차량이 스핀하지 않고 안전하게 정지할 수 있었다.



이처럼 A7 시나리오는 개별 제어기의 성능뿐만 아니라, 모든 제어기가 조화롭게 통합되어 차량의 비선형 거동을 제어했을 때 나타나는 성능 향상을 가장 명확하게 증명하는 사례이다.

### 5.2 가장 부족했던 시나리오
A4 정상선회에서 understeer gradient 가 안 맞았는가? 왜?

가장 부족했던 시나리오는 본 설계에서 가장 KPI 지표가 기대에 미치지 못했던 시나리오는 A4(정상 선회, Steady-state circular driving)이다. 실험 결과, 베이스라인 대비 understeerGradient가 이론적 모델에서 정의한 값과 편차를 보였으며, 이는 차량이 설계자의 의도보다 약간 더 언더스티어 성향을 띠거나 선회 응답성이 저하됨을 의미한다. 
- 가설 1: 비선형 타이어 거동과 선형 모델 간의 괴리

제어기 설계에 사용된 Bicycle Model은 타이어의 횡력이 슬립 앵글에 선형적으로 비례한다고 가정한다 (). 그러나 실제 시뮬레이터(14DOF)는 Pacejka의 Magic Formula를 기반으로 하는 비선형 타이어 모델을 사용한다. A4 시나리오와 같은 정상 선회 상황에서는 횡가속도가 점진적으로 증가하면서 타이어의 횡력이 포화 즉, Saturation 영역에 진입하게 되는데, 선형 모델 기반으로 튜닝된 제어기는 이러한 비선형성을 충분히 보상하지 못하여 언더스티어 구배의 오차를 유발하였다.


- 가설 2: 종방향/횡방향 동역학의 결합 효과

이론적 2자유도 모델은 차량의 종방향 속도가 일정하다고 가정하고 횡방향 제어를 분리(Decoupling)하였다.하지만 실제 14DOF 모델에서는 선회 시 발생하는 롤링(Rolling) 모션에 따른 하중 이동(Load Transfer)이 발생하며, 이는 전후륜 타이어의 수직 하중(Fz)을 변화시킨다. 수직 하중이 변하면 타이어의 코너링 강성()도 함께 변하게 되는데, 설계 모델은 이러한 동적 하중 변화를 고려하지 않았기 때문에 정상 상태 선회 시 발생하는 언더스티어 구배의 변화를 정확히 추종하지 못한 것으로 판단된다.


분석 요약:본 제어기는 극한 상황(A1, A7 등)에서의 자세 제어에는 매우 강건한 성능을 보였으나, A4와 같은 정밀한 정상 상태 선회 제어에서는 선형 모델의 한계로 인해 미세한 동역학적 오차가 발생하였다. 이는 제어기 튜닝 시 사용한 모델의 단순화 과정에서 발생한 필연적인 결과로 보이며, 향후 비선형 타이어 모델을 반영한 Gain Scheduling의 정교화가 필요함을 알려준다.




### 5.3 만약 더 시간이 있었다면
이번 프로젝트를 진행하며 제어 성능을 보완하고 싶은 부분들이 몇 가지 있었다. 시간에 쫓겨 완벽하게 구현하지 못한 점들이 아쉬움으로 남는다.



노면 상황에 따른 실시간 게인 튜닝: 현재 제어기는 고정된 게인을 사용하고 있어 마른 노면에서는 잘 작동하지만, 젖은 노면이나 눈길 같은 다양한 환경에서는 성능이 다소 떨어질 수 있을 것 같다. 시간이 좀 더 있었다면 노면의 마찰 계수를 실시간으로 추정해서 제어기 게인이 스스로 최적값을 찾아가는 '적응형 제어'를 시도해 보고 싶었다.



에너지 효율을 고려한 제동 제어: 현재는 제동 성능을 최우선으로 하여 ABS나 ESC를 공격적으로 작동시켰는데, 실제 차량에서는 브레이크 패드의 마모나 에너지 소모도 중요한 요소일 것이다. 단순히 거리 단축뿐만 아니라, 차량의 거동을 해치지 않는 선에서 불필요한 제동 개입을 줄이고 최적의 토크만 사용하는 방법을 고민해보고 싶다.



비선형 모델을 반영한 제어기 개선: 지금은 설계를 단순화하려고 선형 모델을 썼는데, A4 시나리오처럼 선회 성능이 중요한 구간에서는 비선형적인 타이어 특성이 결과값에 꽤 큰 영향을 주는 것 같다. 기말고사 이후나 방학 때 여유가 된다면, 타이어의 비선형 특성을 더 정교하게 반영한 모델을 적용해보고 싶다는 생각이 들었다.



사실 완벽한 제어기를 만드는 게 쉽지 않다는 걸 이번 과제를 통해 뼈저리게 느꼈다. 그래도 부족한 부분을 스스로 고민해보고 분석했다는 점에서 이번 프로젝트가 큰 공부가 되었다고 생각한다.
